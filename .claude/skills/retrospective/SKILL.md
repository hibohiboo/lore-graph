# 振り返りレポート生成

以下の手順でコードベースを探索し、振り返りレポートを `docs/` に保存してください。

## 手順

1. **コードベース探索**
   - ディレクトリ構成・各 app/package の役割を把握する
   - 主要な技術スタックと依存パッケージを確認する
   - API ルート、データモデル、フロントエンドコンポーネントを読む
   - テスト構成を確認する
   - git ログで直近の変更を確認する (`git log --oneline -20`)

2. **レポート生成**
   ファイル名: `docs/retrospective-YYYY-MM-DD.md`（今日の日付を使う）

   以下のセクションを含めること：
   - **概要**: プロジェクトが何をするものか一言で
   - **作ったもの**: アーキテクチャ図・モノレポ/パッケージ構成表
   - **技術選定の判断**: 各技術を選んだ理由（なぜ他ではなくこれか）
   - **実装して良かったこと**: 設計判断として正解だったもの
   - **苦労したこと・課題**: 踏んだ問題と教訓
   - **今後やること**: 優先度付きタスク表

3. **出力**
   - ファイルに保存後、保存パスをユーザーに伝える
   - レポートの「苦労したこと」と「今後やること」を簡潔にサマリーする

## 注意事項

- コードから読み取れる事実に基づいて書く（推測を混ぜない）
- 日本語で書く
- 既存の `docs/mvp-retrospective.md` がある場合は参照して差分・進捗を反映する
